🎯 Offensive Fuzzing Capabilities

Advanced attack generation for authorized security research

💉

Injection Attacks

SQLi, NoSQLi, Command Injection, LDAP, XPath

  • Time-based blind SQLi payloads
  • NoSQL operator injection ($ne, $gt, etc.)
  • Command injection (backticks, pipes, &&)
  • LDAP filter bypass techniques
🔓

Authentication Bypass

JWT manipulation, session fixation, auth logic flaws

  • JWT algorithm confusion (alg: none)
  • Session token prediction
  • OAuth redirect hijacking
  • Race condition exploits
📂

Path Traversal

Directory traversal, LFI, RFI, file upload bypass

  • ../../../ encoding variations
  • Null byte injection (%00)
  • Double URL encoding
  • MIME type bypass techniques
💥

Buffer Overflow

Integer overflow, heap spray, format string attacks

  • Integer wraparound payloads
  • Heap spray patterns
  • Format string vulnerabilities (%n, %x)
  • Stack smashing techniques
🔐

Cryptographic Attacks

Weak cipher exploitation, padding oracle, timing attacks

  • CBC padding oracle detection
  • ECB mode exploitation
  • Timing attack vectors
  • Known plaintext attacks
🌐

Protocol Abuse

HTTP smuggling, SSRF, DNS rebinding, cache poisoning

  • HTTP request smuggling (CL.TE, TE.CL)
  • SSRF with localhost bypass
  • DNS rebinding payloads
  • Cache poisoning headers

🛡️ Built-In Safety Guardrails

Mandatory authorization and scope enforcement at every layer

🔒

Scope Gates

Automatic target validation before any fuzzing

  • IP/domain allowlist enforcement
  • CIDR range validation
  • Local network blocking (127.0.0.1, etc.)
  • Scope file required (cactusfuzz_scope.txt)
📋

Authorization Checks

Mandatory authorization file with expiration dates

  • Signed authorization document required
  • Date/time range validation
  • Authorized personnel verification
  • Audit trail generation
🎯

Proposal Mode

Generate attack payloads without execution

  • --proposal-only flag (recommended)
  • Review payloads before sending
  • Human-in-the-loop approval
  • Export to JSON for analysis

⌨️ Safe Lab Usage Example

Always use --proposal-only for review-first workflows
