---

### CVTStaking.sol
**Size:** ~180 lines  
**Purpose:** Staking contract with rewards  
**Features:**
- 10% base APY
- 25% early staker bonus (first 30 days) = 12.5% effective APY
- 100 CVT minimum stake
- Per-second reward calculation
- Claim without unstaking
- Emergency withdrawal
- Requires CVT token address in constructor
- Needs minter role granted by token contract

**Formula:** `reward = (stakedAmount × REWARD_RATE × stakingDuration) / (365 days × 100)`

---

### CVTGovernance.sol
**Size:** ~250 lines  
**Purpose:** DAO governance with proposals and voting  
**Features:**
- 10,000 CVT proposal threshold
- 7-day voting period
- 2-day execution timelock
- 10% quorum (10M CVT out of 100M)
- Multiple proposal types (FeatureRequest, BudgetAllocation, Partnership, TreasuryManagement, ProtocolUpgrade, Other)
- Proposal states (Pending → Active → Defeated/Succeeded → Queued → Executed/Cancelled)
- 1 CVT = 1 vote (voting power based on token balance at proposal creation)
- Owner executes approved proposals after timelock

**Voting Power:** Snapshot at proposal creation block, prevents double-voting

### scripts/deploy.js
**Size:** ~200+ lines  
**Purpose:** Automated 3-contract deployment with setup  
**Steps:**
1. Deploy CVTToken
2. Mint 30M CVT to deployer (for liquidity, staking, governance)
3. Deploy CVTStaking with token address
4. Deploy CVTGovernance with token address
5. Grant minter role to staking contract
6. Save deployment addresses to JSON file
7. Output PolygonScan verification commands
8. Generate .env format for frontend

**Output:** Deployment addresses + verification commands + .env snippet
